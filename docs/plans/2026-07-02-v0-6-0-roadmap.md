## Slice 1 — Lucifero chairman reale

**File principale:** `packages/core/src/agents/councilApi.ts:528-532`

**Stato attuale:**
```ts
if (chairman && !completedIds.has(chairman.id)) {
  callbacks.onSynthesisStart?.();
  callbacks.onSynthesisDone?.('Lucifero synthesis: see agent outputs above.');
  // ... emits member_cost with durationMs: 0 ...
}
```

**Stato target:**

Lucifero riceve come input:
1. Il task originale dell'utente.
2. Gli output dei 5 specialisti (Caronte, Nettuno, Gerione, Plutone, Minosse).
3. Il proprio systemPrompt (già definito in `roles.ts:181-`).

Lucifero esegue `AgentHarness.run()` con:
- `model`: stesso modello del council.
- `messages`: `[system(lucifero), user(ricostruito con tutti gli output)]`.
- `tools`: solo workspace tools (stessi dei specialisti).
- `providerStream`: stesso del council.

Lucifero emette eventi:
- `agent_start` con `memberId: 'lucifer'`, `memberName: 'Lucifero'`.
- `message_delta` × N (streaming, real-time).
- `message_end` (full synthesis).
- `agent_end` con `reason: 'completed' | 'error' | 'cancelled'`.

Lucifero callback:
- `onSynthesisStart()` quando inizia.
- `onSynthesisChunk(delta)` per ogni delta (così la TUI può fare
  typewriter effect).
- `onSynthesisDone(content, toolCalls, thinking)` alla fine.

**Acceptance criteria:**

- `dispatchCouncil(...).emit('lucifer_synthesis', ...)` non più vuoto.
- ChatStream mostra `· Lucifero` (in purple, colore #8b5cf6) per
  il messaggio finale.
- Headless mode (`--headless --council --task X`) emette
  `memberName: "Lucifero"` nel NDJSON.
- E2E test: stub provider che risponde a 6 invocazioni
  (5 specialisti + 1 chairman) — assert tutti i delta del
  chairman sono in output.

**Test aggiuntivi (~12 nuovi):**
- `council-chairman.test.ts`:
  - chairman emette agent_start con memberId corretto.
  - chairman emette almeno 1 message_delta.
  - chairman emette message_end con content non-vuoto.
  - chairman emette agent_end con reason.
  - chairman error propaga exit code (no swallow).
  - chairman vuoto (nessun specialist) → chairman non gira.

**Slice 1.5 — Robustness:**
- Se chairman fallisce (API error), il council non è perso: gli
  output dei 5 specialisti restano disponibili come fallback
  synthesis.
- Se chairman emette 0 delta, log warn + usa ultima message
  accumulata (no silent fail).

## Slice 3 — Grounding migliore del council

**Problema:** oggi, dopo che i 5 specialisti e Minosse hanno
parlato, il "verdict" è first-opinion-wins (in molte skill) o
vuoto (in council raw). Serve un protocollo esplicito.

**Proposta: `groundCouncil(agentOutputs: AgentOutput[]): GroundedDecision`**

```ts
interface GroundedDecision {
  position: string;          // la posizione "vincitrice"
  confidence: 'high' | 'medium' | 'low';
  dissent: Array<{           // opinioni in disaccordo
    agent: string;
    reason: string;
  }>;
  rationale: string;         // perché questa posizione
}
```

**Strategia di grounding (3 livelli):**

1. **Unanimità** (5/5 specialisti d'accordo) → confidence: high.
2. **Maggioranza** (3+/5 d'accordo, con 1-2 dissenting) → confidence: medium, dissent elencato.
3. **Disaccordo** (<3/5 d'accordo) → confidence: low, Lucifero DEVE esplicitamente scegliere + motivare.

**Implementazione:**
- Nuovo helper in `packages/core/src/council/grounding.ts`.
- Chiama un piccolo LLM (o usa Lucifero stesso) per classificare
  la posizione di ciascun agente su una scala binaria
  (agree / disagree) rispetto a una sintesi preliminare.
- Output passato a Lucifero come context addizionale.

**Acceptance:**
- `groundCouncil` ritorna sempre un oggetto valido (no null).
- 5 unit test sui 3 scenari (unanimità, maggioranza, disaccordo).
- In `dispatchCouncil`, il prompt del Lucifero include la
  sezione `GROUNDING:` con il verdetto.

**Costo:** +1 piccola chiamata LLM. Mitigation: opzionale
(flag `--no-grounding` per saltare in CI, dove conta la velocità).

| Rischio | Mitigation |
|---|---|
| Chairman synthesis troppo verbosa → overwhelming UI | `--no-synthesis` flag per nascondere in modalità minimal |
| Lucifero chiama LLM con context troppo grande | Truncation esplicita: max 4KB per specialist output passato a Lucifero |
